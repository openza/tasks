Openza Tasks syncs Microsoft To Do through Microsoft Graph using MSAL.NET with Windows Web Account Manager (WAM).

<Aside type="caution">
  Microsoft To Do integration requires a Microsoft personal or work/school account.
</Aside>

## Setup Process

<Steps>
1. **Open Settings** in Openza Tasks

2. **Confirm the Microsoft app client ID**

   Store builds can include this automatically. Source builds can set `OPENZA_TASKS_MSTODO_CLIENT_ID`, pass `-p:MicrosoftToDoClientId=...` at build time, or paste the public client ID in Settings.

   For source builds, the Azure app registration must be a public client/native app and include this redirect URI:

   ```text
   ms-appx-web://microsoft.aad.brokerplugin/{client_id}
   ```

3. **Click Connect Microsoft To Do**

4. **Sign in with Microsoft**

   Windows shows the account broker sign-in experience and may reuse a Windows account already connected to the device.

5. **Run Sync**
</Steps>

## What Syncs

| Item | Synced |
|------|--------|
| Tasks | Yes |
| Lists | Yes |
| Due Dates | Yes |
| Complete/Reopen | Yes |
| Create/Edit/Delete write-back | Not in V1 |
| Recurrence | Not yet |
| Categories | Yes, imported as labels when available |

## Troubleshooting

### Client ID missing

Open-source source builds need a public Azure app registration client ID. This is not a client secret.

### Broker redirect URI missing

If sign-in fails immediately after choosing an account, verify that the Azure app registration includes `ms-appx-web://microsoft.aad.brokerplugin/{client_id}` as a mobile and desktop redirect URI.

### Sign-in or refresh issues

If Microsoft sync stops:
1. Disconnect your account in Settings
2. Connect Microsoft To Do again
3. Run Sync

<Aside type="tip">
  The current access token is stored in Windows Credential Locker. The MSAL cache is stored locally and encrypted for the current Windows user.
</Aside>

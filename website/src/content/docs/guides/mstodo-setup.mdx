Openza Tasks syncs Microsoft To Do through Microsoft Graph using MSAL.NET. Windows Web Account Manager (WAM) is used when available, with browser sign-in as a fallback.

<Aside type="caution">
  Microsoft To Do integration requires a Microsoft personal or work/school account.
</Aside>

## Setup Process

<Steps>
1. **Open Settings** in Openza Tasks

2. **Click Connect Microsoft To Do**

3. **Choose the Microsoft account for this feature**

   Openza uses its built-in public Microsoft app registration. You do not need to paste a client ID or configure Azure.

4. **Sign in with Microsoft**

   Windows may show the native account broker. If broker sign-in is unavailable, Openza can fall back to browser sign-in.

5. **Run Sync**
</Steps>

## What Syncs

| Item | Synced |
|------|--------|
| Tasks | Yes |
| Lists | Yes |
| Due Dates | Yes |
| Complete/Reopen | Yes |
| Create/Edit/Delete write-back | Not in V1 |
| Recurrence | Not yet |
| Categories | Yes, imported as labels when available |

## Troubleshooting

### Sign-in or refresh issues

If Microsoft sync stops:
1. Disconnect your account in Settings
2. Connect Microsoft To Do again
3. Run Sync

<Aside type="tip">
  Microsoft To Do and OneDrive backup can use different Microsoft accounts. Microsoft tokens are kept in the local MSAL cache encrypted for the current Windows user.
</Aside>

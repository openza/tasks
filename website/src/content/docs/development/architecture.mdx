Openza Tasks is a Windows-native app with a small UI project and a testable core.

## Project Structure

```text
src/
  Openza.Tasks/
    App.xaml                 app startup, schema upgrade, service wiring
    MainWindow.xaml          thin window host
    Shell/                   navigation shell and app-level workflows
    Pages/                   settings and page-level surfaces
    Controls/                reusable task, project, detail, and state controls
    Services/                Windows-specific services

  Openza.Tasks.Core/
    Data/                    SQLite schema and repositories
    Migration/               app data and backup path helpers
    Models/                  tasks, projects, labels, integrations
    Services/                backup and Markdown import/export
    Sync/                    C# provider sync engine
    Credentials/             credential-store abstraction

  Openza.Tasks.Tests/
    Unit, repository, backup, and sync tests
```

## Key Technologies

| Layer | Technology |
|-------|------------|
| UI | WinUI 3 |
| Runtime | .NET 10 |
| Windows SDK | Windows App SDK 2.0.x |
| Database | SQLite via Microsoft.Data.Sqlite |
| Sync | C# provider adapters and orchestration |
| Credentials | Windows Credential Locker and encrypted MSAL cache |
| Tests | xUnit |

## Data Flow

```text
WinUI shell -> Openza.Tasks.Core services -> SQLite/provider adapters
```

The active sync engine is C#. Historical Rust FFI work is preserved in repository history and older releases, and is not shipped with the WinUI app. Store V1 is a fresh-start WinUI product and does not auto-migrate old local app stores.

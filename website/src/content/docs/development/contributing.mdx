We welcome contributions to Openza Tasks. The active app is Windows-native WinUI 3.

## Getting Started

<Steps>
1. Fork the repository on GitHub.
2. Clone your fork:

   ```powershell
   git clone https://github.com/YOUR_USERNAME/tasks.git
   cd tasks
   ```

3. Create a branch:

   ```powershell
   git switch -c feature/your-feature-name
   ```

4. Restore, test, and build:

   ```powershell
   dotnet restore Openza.Tasks.slnx
   dotnet test src\Openza.Tasks.Tests\Openza.Tasks.Tests.csproj -c Release
   dotnet build src\Openza.Tasks\Openza.Tasks.csproj -c Release -p:Platform=x64 --no-restore
   ```

5. Submit a pull request.
</Steps>

## Coding Standards

- Use native WinUI controls and Fluent spacing.
- Keep data, sync, backup, import/export, and provider logic in `Openza.Tasks.Core`.
- Add tests for sync, provider mapping, backup, import/export, schema upgrade, and repository changes.
- Do not commit secrets, generated certificates, MSIX packages, or Store-private data.

## Pull Request Guidelines

<Aside type="tip">
  Include screenshots for UI changes and mention any migration behavior.
</Aside>

- Describe what changed and why.
- Keep PRs focused on a single feature or fix.
- Ensure the app project builds, not only the core library.
- Run secret scanning before publishing public branches.

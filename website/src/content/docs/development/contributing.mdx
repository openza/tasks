# Contributing

We welcome contributions to Openza Tasks! Here's how to get started.

## Getting Started

<Steps>
1. **Fork the repository** on GitHub

2. **Clone your fork**
   ```bash
   git clone https://github.com/YOUR_USERNAME/tasks.git
   cd tasks
   ```

3. **Create a branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```

4. **Install dependencies**
   ```bash
   flutter pub get
   ```

5. **Make your changes** following our coding standards

6. **Regenerate code** (if you modified models or providers)
   ```bash
   dart run build_runner build --delete-conflicting-outputs
   ```

7. **Test your changes**
   ```bash
   flutter analyze
   flutter test
   ```

8. **Submit a pull request**
</Steps>

## Coding Standards

- Follow the [Dart Style Guide](https://dart.dev/guides/language/effective-dart/style)
- Use `flutter analyze` to check for issues
- Write tests for new features
- Keep commits atomic and well-described

## Pull Request Guidelines

<Aside type="tip">
  Link any related issues in your pull request description.
</Aside>

- Describe what your PR does and why
- Include screenshots for UI changes
- Ensure all tests pass
- Keep PRs focused on a single feature or fix

## Reporting Issues

Found a bug? Have a feature request?

1. Search [existing issues](https://github.com/openza/tasks/issues) first
2. If not found, create a new issue with:
   - Clear description
   - Steps to reproduce (for bugs)
   - Expected vs actual behavior
   - System information

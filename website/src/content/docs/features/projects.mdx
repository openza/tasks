Projects help you group related tasks together inside the selected Openza space. In WinUI V1, Openza projects are the user-facing project model; provider project and list names remain source context on imported tasks.

## Openza Projects

Projects created in Openza Tasks are local and fully controlled by Openza:

- Create, edit, and delete freely
- Move tasks between them
- Use them to filter the current task list

## Provider Project Context

Provider project or list names from Todoist and Microsoft To Do are preserved as source context:

- They help you understand where a source-linked task came from
- They do not replace Openza projects
- Provider project/list management stays in the original provider

<Aside type="note">
When you add a provider source item to Inbox, you can assign the resulting Openza task to any local Openza project.
</Aside>

### The Inbox

Inbox is the default place for unprojected tasks:

- Tasks without a project go here
- Synced source items can be reviewed before becoming Openza tasks
- Select a project later from task details

## Creating a Project

<Steps>
1. Click **New project** in the command bar
2. Enter a project name
3. Click **Create**
</Steps>

The new project appears in the Projects pane for the selected space.

## Editing a Project

<Steps>
1. Find the project in the Projects pane
2. Click **Edit**
3. Change project details supported by the current app
4. Click **Save**
</Steps>

<Aside type="note">
Provider project and list names are source context. Edit them in Todoist or Microsoft To Do, then sync.
</Aside>

## Deleting a Project

<Steps>
1. Select the project
2. Click **Delete**
3. Confirm the deletion
</Steps>

<Aside type="caution">
Deleting a project cannot be undone. Move tasks to another project or Inbox first if you want to keep the project grouping.
</Aside>

## Moving Tasks Between Projects

You can move tasks to a different project in two ways:

### From Task Details

<Steps>
1. Select a task to open the details pane
2. Use the project dropdown to select a new project
3. Click **Save task**
</Steps>

## Filtering by Project

Click any project in the Projects pane to filter the task list:

- Only tasks from that project are shown
- The project name appears in the header
- Select **All projects** or another view to clear the filter

## Spaces

Projects belong to the selected Openza space. Spaces are local visibility scopes, not provider accounts. Use Settings > Spaces to manage the spaces available in the app.

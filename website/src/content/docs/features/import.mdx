Import tasks directly from Markdown files using GitHub Flavored Markdown (GFM) checkbox syntax.

## Supported Format

Openza Tasks recognizes GFM task list syntax:

```markdown
- [ ] Open task (will be imported as active)
- [x] Completed task (will be imported as done)
```

### What Gets Imported

- **Task title** - The text after the checkbox
- **Completion status** - Based on `[ ]` (open) or `[x]` (completed)

<Aside type="note">
Priority, due dates, and labels are not imported from markdown. You can add these after import.
</Aside>

## Import Methods

### Import File

<Steps>
1. Click **Import** in the command bar
2. Select a `.md`, `.markdown`, or `.txt` file
3. Openza Tasks imports detected checkboxes into the Inbox project
</Steps>

## Duplicate Detection

Duplicate detection is not part of WinUI V1. Review imported tasks after import and delete anything you do not need.

<Aside type="tip">
Import into a clean Inbox or project-specific file when you want an easier review pass.
</Aside>

## Safety Limits

To prevent issues with large files:

- **Maximum line length**: 10,000 characters
- **Encoding**: Windows text picker default decoding

## Examples

### Simple Task List

```markdown
- [ ] Buy milk
- [ ] Call dentist
- [x] Pay bills
```

### Mixed Content (Only Checkboxes Imported)

```markdown
# Shopping List

## Groceries
- [ ] Eggs
- [ ] Bread
- [x] Butter

## Notes
This is just a note and won't be imported.
```

Only the items with `- [ ]` or `- [x]` prefix are imported as tasks.

## Use Cases

### From Meeting Notes
Copy action items from meeting notes directly into Openza Tasks.

### From Other Apps
Export tasks from another tool as markdown, then import here.

### Quick Entry
Write tasks in your favorite text editor, then import in bulk.

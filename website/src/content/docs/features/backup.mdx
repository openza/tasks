Openza Tasks can automatically back up your local SQLite database to protect your tasks from data loss. Local durable backups are the foundation; OneDrive backup is an optional extra layer.

## Automatic Backups

By default, Openza Tasks creates one startup backup per day. You can turn startup backups on or off in Settings.

### Enabling/Disabling Auto-Backup

<Steps>
1. Open **Settings** from the left navigation
2. Select **Backups**
3. Toggle **Create a backup on startup** on or off
</Steps>

## Manual Backup

Create a backup anytime from Settings:

<Steps>
1. Open **Settings**
2. Click **Create backup**
3. The backup appears in your backup list
</Steps>

## Backup Management

### Viewing Backups

Your backups are displayed with:
- **Reason** such as daily, manual, legacy, pre-restore, pre-import, or pre-migration
- **Date and time** of creation
- **Task, project, and space counts**
- **File size**

### Export Backup

Export a backup to save externally or transfer to another computer:

<Steps>
1. Select the backup you want to export
2. Click **Export selected**
3. Choose a save location
4. The backup is saved as a `.db` file
</Steps>

### Restore from Backup

<Aside type="caution">
Restoring a backup will replace your current data. This action cannot be undone.
</Aside>

<Steps>
1. Select the backup to restore
2. Click **Restore selected**
3. Confirm the restore dialog
</Steps>

Openza Tasks creates a safety copy of the current database before replacing it.

### Restore from External File

Restore a backup from another computer or external storage:

<Steps>
1. Open **Settings**
2. Click **Restore from file**
3. Select a `.db` file from your computer
4. Confirm the restore dialog
</Steps>

<Aside type="note">
External backups are restored directly. Openza still creates a safety backup before replacing the live database.
</Aside>

## Backup Storage

- Backups are stored in a stable Openza backup folder outside Windows package `LocalState`
- Production uses `%LOCALAPPDATA%\Openza\Tasks\Backups`
- Dev uses `%LOCALAPPDATA%\Openza\Tasks Dev\Backups`
- Production and dev builds do not share backup folders
- Retention keeps recent daily, weekly, monthly, and event backups
- Older package-local backups are copied forward when found, but not deleted automatically

## OneDrive Backup

OneDrive backup is optional and disabled by default. When enabled, Openza uploads backup snapshots to its app folder in the Microsoft account you choose for OneDrive backup.

<Aside type="note">
OneDrive backup is cloud backup, not multi-device live sync. Your local SQLite database remains the primary database.
</Aside>

You can enable passphrase encryption so backup files are encrypted before upload. On a new PC, encrypted OneDrive backups require the passphrase.

### OneDrive Actions

In Settings > Backups, you can:

- Enable or disable OneDrive backup
- Choose or change the Microsoft account used for OneDrive backup
- Upload a backup now
- Refresh the cloud backup list
- Restore from a selected OneDrive backup
- Add, change, or remove the optional passphrase

Deleting a local backup does not delete the cloud copy. Cloud pruning only removes Openza-owned backup files according to the configured retention policy.

## Best Practices

- Keep auto-backup enabled for peace of mind
- Enable OneDrive backup if you want recovery after PC loss, reset, or disk failure
- After major changes, create a manual backup
- Before restoring, Openza creates a safety backup of the current database

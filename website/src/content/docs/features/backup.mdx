Openza Tasks can automatically back up your local database to protect your tasks from data loss.

## Automatic Backups

By default, Openza Tasks creates one startup backup per day. You can turn startup backups on or off in Settings.

### Enabling/Disabling Auto-Backup

<Steps>
1. Open **Settings** from the left navigation
2. Find the **Backups** section
3. Toggle **Create a backup on startup** on or off
</Steps>

## Manual Backup

Create a backup anytime from Settings:

<Steps>
1. Open **Settings**
2. Click **Create backup**
3. The backup appears in your backup list
</Steps>

## Backup Management

### Viewing Backups

Your backups are displayed with:
- **Date** shown in calendar format (day and month)
- **Time** of creation
- **File size**

### Export Backup

Export a backup to save externally or transfer to another computer:

<Steps>
1. Select the backup you want to export
2. Click **Export selected**
3. Choose a save location
4. The backup is saved as a `.db` file
</Steps>

### Restore from Backup

<Aside type="caution">
Restoring a backup will replace your current data. This action cannot be undone.
</Aside>

<Steps>
1. Select the backup to restore
2. Click **Restore selected**
3. Confirm the restore dialog
</Steps>

Openza Tasks creates a safety copy of the current database before replacing it.

### Restore from External File

Restore a backup from another computer or external storage:

<Steps>
1. Open **Settings**
2. Click **Restore from file**
3. Select a `.db` file from your computer
4. Confirm the restore dialog
</Steps>

<Aside type="note">
External backups are restored directly without being added to your backup list.
</Aside>

## Backup Storage

- Backups are stored in your app data folder
- Up to **7 backups** are retained automatically
- Oldest backups are deleted when the limit is reached

## Best Practices

- Keep auto-backup enabled for peace of mind
- Download important backups to external storage
- After major changes, create a manual backup
- Before restoring, download your current data first
